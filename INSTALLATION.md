# Подробная инструкция по установке и запуску проекта Litera

## Шаг 1: Проверка установки Python

1. Откройте командную строку (PowerShell или CMD).
2. Проверьте, установлен ли Python, выполнив команду:
   ```
   python --version
   ```
   Или:
   ```
   python3 --version
   ```
3. Если Python не установлен, скачайте и установите его с официального сайта: https://www.python.org/downloads/
   - При установке обязательно отметьте галочку "Add Python to PATH"

## Шаг 2: Переход в папку проекта

1. Откройте командную строку (PowerShell или CMD).
2. Перейдите в папку проекта. Например, если проект находится в `C:\Users\darke\alltoall\Litera\Myproject`, выполните:
   ```
   cd C:\Users\darke\alltoall\Litera\Myproject
   ```
   Или если вы уже находитесь в папке `Litera`, выполните:
   ```
   cd Myproject
   ```

## Шаг 3: Создание виртуального окружения

1. Убедитесь, что вы находитесь в папке `Myproject` (там, где находится файл `manage.py`).
2. Создайте виртуальное окружение командой:
   ```
   python -m venv venv
   ```
   Или если команда `python` не работает, попробуйте:
   ```
   python3 -m venv venv
   ```
   Или:
   ```
   py -m venv venv
   ```
3. Дождитесь завершения создания виртуального окружения. В папке `Myproject` появится новая папка `venv`.

## Шаг 4: Активация виртуального окружения

### Для PowerShell:
Выполните команду:
```
.\venv\Scripts\Activate.ps1
```

Если появится ошибка о политике выполнения скриптов, сначала выполните:
```
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
Затем снова выполните:
```
.\venv\Scripts\Activate.ps1
```

### Для обычной командной строки (CMD):
Выполните команду:
```
venv\Scripts\activate.bat
```

### Как понять, что виртуальное окружение активировано:
После активации в начале строки командной строки появится префикс `(venv)`, например:
```
(venv) C:\Users\darke\alltoall\Litera\Myproject>
```

## Шаг 5: Установка зависимостей проекта

1. Убедитесь, что виртуальное окружение активировано (видите префикс `(venv)` в командной строке).
2. Установите все необходимые пакеты из файла `requirements.txt`:
   ```
   pip install -r requirements.txt
   ```
3. Дождитесь завершения установки всех пакетов. Будут установлены:
   - Django 4.2.26
   - asgiref 3.10.0
   - sqlparse 0.5.3
   - tzdata 2025.2

## Шаг 6: Применение миграций базы данных

1. Убедитесь, что виртуальное окружение все еще активировано.
2. Примените миграции базы данных:
   ```
   python manage.py migrate
   ```
   Или если команда `python` не работает:
   ```
   python3 manage.py migrate
   ```
   Или:
   ```
   py manage.py migrate
   ```
3. Дождитесь завершения выполнения миграций. Будет создана база данных `db.sqlite3` (если её еще нет).

## Шаг 7: Создание суперпользователя (опционально, но рекомендуется)

1. Убедитесь, что виртуальное окружение все еще активировано.
2. Создайте учетную запись администратора:
   ```
   python manage.py createsuperuser
   ```
3. Введите имя пользователя, email (опционально) и пароль по запросу.
   - Это позволит вам заходить в административную панель Django.

## Шаг 8: Запуск сервера разработки

1. Убедитесь, что виртуальное окружение все еще активировано.
2. Запустите сервер разработки Django:
   ```
   python manage.py runserver
   ```
3. Вы увидите сообщение примерно такого вида:
   ```
   Starting development server at http://127.0.0.1:8000/
   Quit the server with CTRL-BREAK.
   ```
4. Откройте браузер и перейдите по адресу: http://127.0.0.1:8000/
5. Если вы создали суперпользователя, административная панель доступна по адресу: http://127.0.0.1:8000/admin/

## Шаг 9: Остановка сервера

1. Чтобы остановить сервер, нажмите `CTRL+C` в командной строке, где запущен сервер.

## Шаг 10: Деактивация виртуального окружения (когда закончите работу)

1. Когда закончите работу с проектом, деактивируйте виртуальное окружение:
   ```
   deactivate
   ```
2. Префикс `(venv)` исчезнет из командной строки.

## Важные замечания

- **Всегда активируйте виртуальное окружение перед работой с проектом!** Без активации команды Django не будут работать.
- Если вы закрыли командную строку, вам нужно будет снова активировать виртуальное окружение (Шаг 4).
- Виртуальное окружение нужно создавать только один раз. После этого просто активируйте его каждый раз при работе с проектом.
- Папку `venv` не нужно добавлять в систему контроля версий (git), она уже должна быть в `.gitignore`.

## Решение возможных проблем

### Проблема: "python не является внутренней или внешней командой"
**Решение:** Установите Python и убедитесь, что он добавлен в PATH, или используйте `py` вместо `python`.

### Проблема: "pip не является внутренней или внешней командой"
**Решение:** Убедитесь, что виртуальное окружение активировано, или установите pip отдельно.

### Проблема: "ModuleNotFoundError: No module named 'django'"
**Решение:** Убедитесь, что виртуальное окружение активировано и вы установили зависимости из `requirements.txt`.

### Проблема: Ошибка при активации в PowerShell
**Решение:** Выполните `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` и попробуйте снова.
